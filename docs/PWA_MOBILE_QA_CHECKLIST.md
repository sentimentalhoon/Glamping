# PWA Mobile QA Checklist

## 목적
- Android(Chrome), iOS(Safari)에서 설치/실행/오프라인/폼 동작을 검증한다.
- 광고 유입 상황에서 리드 누락이 없는지 확인한다.

## 테스트 환경
- Android: 최신 Chrome
- iOS: 최신 Safari
- 네트워크: Wi-Fi, LTE/5G, 오프라인 모드
- 배포 환경: 실제 HTTPS 도메인(개발 localhost 제외)

## 사전 조건
- `NEXT_PUBLIC_SITE_URL` 실도메인 설정 완료
- `LEAD_WEBHOOK_URL` 설정 완료(또는 fallback 로그 확인 가능)
- 최신 배포 반영 완료
- 브라우저 캐시 삭제 후 시작

## 공통 체크 (Android/iOS)
1. 홈 접속
- 항목: 첫 로딩, 주요 섹션 렌더링, CTA 버튼, 지도 fallback 문구
- 기대 결과: 레이아웃 깨짐 없음, 주요 버튼 클릭 가능

2. 리드 폼 제출 (온라인)
- 항목: `/`의 문의 폼, `/lp/membership-a`, `/lp/membership-b` 폼 제출
- 기대 결과: 성공 메시지 노출, 웹훅/로그에 데이터 수신
- 확인 필드: `name`, `phone`, `interest`, `landingPath`, `utm_*`

3. 리드 폼 제출 (오프라인)
- 항목: 네트워크 차단 후 제출
- 기대 결과: 오프라인 안내 메시지 노출, 앱 크래시 없음

4. 오프라인 페이지
- 항목: 오프라인 상태에서 새 페이지 진입
- 기대 결과: `/offline` 폴백 표시

5. 미디어/성능
- 항목: Hero 영역 로딩, 스크롤 시 프레임 드랍, 이미지 깨짐/과도한 크롭
- 기대 결과: 모바일에서 영상 강제 미사용(정적 이미지), 심한 끊김 없음

## Android 전용 체크 (Chrome)
1. 설치 배너
- 절차: 홈 접속 후 설치 프롬프트 확인
- 기대 결과: "앱 설치" 버튼 노출

2. 설치 동작
- 절차: 설치 버튼 클릭 -> 설치 승인
- 기대 결과: 홈 화면 아이콘 생성, standalone 실행

3. 재실행 동작
- 절차: 홈 화면 아이콘으로 재진입
- 기대 결과: 주소창 없는 앱 형태로 실행

4. 업데이트 반영
- 절차: 새 배포 후 앱 재실행
- 기대 결과: 캐시 갱신 후 최신 UI 표시(강제 새로고침 없이도 반영)

## iOS 전용 체크 (Safari)
1. 설치 안내 배너
- 절차: 홈 접속
- 기대 결과: "홈 화면에 추가" 안내 문구 노출

2. 홈 화면 추가
- 절차: 공유 버튼 -> 홈 화면에 추가
- 기대 결과: 아이콘 생성, 실행 가능

3. standalone 실행
- 절차: 홈 화면 아이콘으로 실행
- 기대 결과: Safari 탭과 분리된 앱 형태 실행

4. 아이콘/타이틀
- 절차: 홈 화면에 생성된 앱 확인
- 기대 결과: 앱 이름/아이콘이 비정상적으로 깨지지 않음

## 광고 유입 QA (필수)
1. UTM 유지
- 절차: `?utm_source=test&utm_medium=cpc&utm_campaign=pwa_qa`로 진입 후 폼 제출
- 기대 결과: 서버 수신 데이터에 UTM 필드 포함

2. A/B 분기
- 절차: `/lp/membership` 10회 진입(시크릿 모드 포함)
- 기대 결과: A/B 분기 동작 확인, 쿠키 기반 고정 동작 확인

## 실패 기준 (배포 차단)
- 폼 제출 성공률 100% 미만(테스트 케이스 기준)
- 오프라인 상태에서 오류/화이트스크린 발생
- 설치 후 실행 불가 또는 즉시 종료
- Android/iOS 중 하나에서 설치 UX 전혀 미동작

- 대규모 광고 시작 전 1회 전체 수행
- 주요 UI 변경/배포 시 간이 회귀 수행
- 월 1회 정기 점검
